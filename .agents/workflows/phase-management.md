### 🟢 Phase Start / In-Progress
1. **Creation**: Create a detailed plan in `docs/planning/phase-[X]-[name].md`.
2. **Context**: Only reference the active phase planning file during development to save tokens.
3. **Drafting**: Use this file for requirements, schema proposals, and implementation steps.

### 🔴 Phase Completion
1. **Move**: Relocate the completed phase file from `docs/planning/` to `docs/history/`.
2. **Log**: Ensure the history file correctly reflects the final implementation details and technical decisions made.
3. **Artifacts**: Clean up any temporary planning artifacts or drafts that are no longer relevant.

### 🚫 Rule
- **No Index File**: Do NOT use `next-planning.md` or any centralized roadmap files to avoid scanning redundancy and token waste.
